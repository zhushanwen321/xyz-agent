1. 包依赖图（分层 + import 规则）

renderer 分层：
  components/panel ──► composables/features ──► stores ──► api/domains
         │                                          │            │
         │                                          ▼            ├──► api/events
         │                                   (api/domains)◄─────┤──► api/transport
         │                                                  └──► api/pending
  api/mock ──► api/events (dispatchSession)

runtime 分层（单向，无环）：
  transport/handlers ──► services ──► services/ports ──► infra
         └──► IMessageBroker (server.ts)

shared 被所有层依赖；runtime ◄──WebSocket──► renderer

新建节点（绿色）：git(domain) / git-executor(port) / git-message-handler / git-service
Import 规则说明
components 不直接 import api/domains统一走 composables/features
stores 间禁止互相 import派生逻辑放 composables/features
runtime services 不直接 import infra必须经 ports（git-status-parser 纯函数例外，见 §5.1）
shared 可被任何层 import禁止反向依赖

2. 关键时序图（入口 → 底层）

F1: git.status 查询 → GitZone 渲染
Panel.onMounted → GitZone.status(sessionId)
  → transport.send('git.status') ──WebSocket──►
    GitMessageHandler → GitService.getStatus(sessionId)
      → SessionService.getSession(sid).cwd
      → IGitExecutor.exec(cwd, 'status', ['--porcelain'])
        → git CLI (execFileSync, 数组参数, timeout:5s)
      → parseGitStatusPorcelain + readGitInfo(branch) + git diff --numstat(stats)
      ◄── GitStatusResult {isRepo, branch, stagedCount, stats, hasConflict, files[]}
    ◄── reply('git.status:result')
  ◄── pending.resolve → GitZone.deriveState(四态 pill + stats + file list)

异常：isRepo=false → 隐藏 GitZone；timeout/git 未安装 → 降级提示
F2: git.stage/unstage/commit 操作（含异常路径）
GitZone 按钮 disabled=true → git.domain.stage(sessionId, paths)
  → transport ──► GitMessageHandler → GitService.stage
    → resolvePaths(cwd, paths) + workspace 白名单校验
    → IGitExecutor.exec('stage', ['add','--',...paths])
    ◄── exitCode 0 → reply('message.status', {status:'staged'})
  ◄── GitZone refresh() → git.status() → button disabled=false

异常分支：
  冲突态 commit    → exitCode 1 → sendError('git_conflict') → toast
  路径越界         → throw SecurityError → sendError('path_not_allowed')
  git 未安装/timeout → execFileSync throw → sendError('git_unavailable') → 降级
  session 不存在    → getSession=undefined → sendError('session_not_found')
F3: Extension 安装多步流（installDir → discovered → finishInstall）
ExtensionPage(tab=dir, input) → extension.installDir(path)
  → transport ──► ExtensionMessageHandler
    → ExtensionService.installLocalDirectory(path)
      → path 白名单(home/tmp) → cpSync → tempDir → discoverExtensions
    ◄── {tempDir, candidates} → reply('extension.discovered')
  ◄── ExtensionDiscoveredPayload → 渲染候选列表

用户选候选 → extension.finishInstall(tempDir, selected)
  → ExtensionService.finishInstall → validate tempDir → cpSync selected→extensions/
    → rmSync tempDir → scanExtensions
  ◄── reply('config.extensions', {extensions}) → onExtensions 订阅刷新

异常：URL/路径非法 → ExtensionInstallError → sendError(code, hint)
F4: compact 触发（含异常路径）
/compact slash → useChat.compact() → chat.domain.compact(sessionId)
  → transport('session.compact') ──► SessionMessageHandler.handleSessionCompact
    → SessionService.ensureActive(sid) → SessionService.compact(sid)
      → MessageDispatcher.compact → broadcast('session.compacting')
      → IPiEngine.compact() → broadcast('session.compacted')
    ◄── reply('session.compacted', {status:'compacted'})

异常：ensureActive 失败 → sendError('compact_failed', 'restore session')
      client 不存在 → throw 'Session not found' → sendError('compact_failed')
      pi engine 错误/超时 → broadcast compacted(error) → sendError('compact_failed')
F5: session.list server-push 订阅
Sidebar.onMounted → useSidebar.loadSessions()
  → events.onGlobalType('session.list', handler)
  ← runtime 侧 session 变更 → RuntimeServer.broadcastSessionList()
    → broadcast({type:'session.list', payload:{groups}})
    ──WebSocket──► transport → events.dispatchGlobal → handler(groups)
  → sessionStore.setGroups → Sidebar re-render（不重载历史）
F7 + F7-UI: chat store 消费补全 + retry/queue 指示位
ServerMessage → events.on(sessionId) → useChat handler
  → ChatStore.appendAssistantChunk → chat-chunk-processor.applyChunk
    switch(msg.type):
      thinking_end    → lastThinking.endTime = Date.now()
      tool_call_update→ find call by toolCallId, update detail
      complete        → fill usage, status=complete/error
      auto_retry_*    → retryStates.set/delete
      queue_update    → queueStates.set/delete
      file_changes    → applyFileChanges(merge by path)
      default         → no-op（记录未知 type）

F7-UI（#13, P1）:
  Composer.onMounted → RetryIndicator v-if="retryState"
    → ChatStore.getRetryState(sid) → 渲染 attempt/maxAttempts
  QueueBubble v-if="queueState" → getQueueState → 渲染 steering/followUp 气泡
  auto_retry_end → retryStates.delete → 消失；message_start → queueStates.delete → 消失
F9 + F10: widget 订阅 + SideDrawer 容器
pi extension setWidget → EventAdapter → extension:widget server-push
  ──WebSocket──► events.dispatchSession(sessionId) → extension.onWidget handler
  → SideDrawer Terminal/Browser tab 更新（1MB 分片 + 1000 行截断）

SideDrawer 触发：GitZone Diff 按钮 → emit('open-side-drawer')
  → Panel → SideDrawer open=true（默认 Terminal tab）→ tab 切换 Terminal/Browser
  → subscribeWidget → onUnmounted 取消订阅
  [不含 Diff tab — Diff 审批排除在 scope 外]
[STALE] message.tool_call_pending / ToolCallStatus.pending 已移除：runtime 不生产该消息（tool 审批链路 Out-of-scope）。详见 code-architecture §3.9。
