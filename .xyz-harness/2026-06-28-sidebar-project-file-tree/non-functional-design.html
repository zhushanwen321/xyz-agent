逐 issue 关键副作用

#1 shared 基础+协议

⚠️ 安全isUnderOrEqual 是越界校验根函数。两类风险：(a) 迁移回归（utils→shared 重写）；(b) symlink 词法绕过（K-1）。缓解：逐字搬移 + 词法单测 + ⑤骨架裁决 realpath。残余：macOS 大小写不敏感⑤验证。
⚠️ 兼容性path-guard import 迁移：所有消费者（git-service+extension-service+file-service）改 import。AC-1.2 grep 兜底 + tsc 编译。

#2 FileService+fs-executor+handler

⚠️ 安全FileService 越界统一守门（file.tree/expand 也校验，NFR-AC-S2）。继承 #1 symlink 词法特性。
⚠️ 性能file.tree 首加载 = 1+M 次 listDir（F-1 修正，M=顶层目录数，monorepo 可达数十），非「2 次」。⑤骨架验证并行化。
⚠️ 稳定性fs 异常→结构化 error。补：(a) fs-executor 超时机制（K-2，node:fs 无内建超时）；(b) symlink 目录逐层展开防护（K-3）。
⚠️ 可观测FileService 缺日志。⑤骨架约束关键操作结构化日志（sessionId/path/reason）。

#3 fileTree store + useFileTree

⚠️ 数据rehydrate（D-019）已删目录路径 graceful 降级 + 跳过日志（D-1）。
⚠️ 性能深层嵌套递归渲染栈深 → ⑥独立 perf Wave（20 层 + 过滤叠加）。
⚠️ 并发K-9 架构约束冲突：AC-3.11 跨 store 失效须在 composable 层（useFileTree watch chat store + 派发 fileTree invalidate），非 store 层。5 个异步竞态（AC-3.7/3.8/3.9/3.10/3.11）进⑤test-matrix。
⚠️ 稳定性overlay 失败降级（AC-3.6 空角标，树仍渲染）。
⚠️ 可观测异步竞态 console.debug 日志 + rehydrate 跳过日志。

#4 FileView+FileTreeRow 重写

⚠️ 性能过滤 debounce（~150ms）+ 深层递归；叠加放大（D-3）：15 层展开 + 过滤输入是 perf Wave 必测场景。

#5 git.diff 全链路
