设计立场

核心计算 = 文件系统读取 + 路径校验编排（技术编排，非业务规则）。 FileService 不引入领域规则——读目录（fs）+ 过滤（.gitignore 纯函数）+ 路径越界校验（shared/path-guard）+ 懒加载编排。沿用三层 + ports，不引入 domain 层（领域贫乏，强行做空壳）。

核心模型

模型类型不变式建模理由
FileNodeDTOpath 唯一；type ∈ {dir,file}；dir 有 children?（懒加载三态）；不含 gitStatus跨进程传输扁平结构。标注独立（D-012）
FileTreeState实体一个 session 一棵树；root=session.cwd；每节点独立加载态4 facet（树缓存/展开/选中/GitOverlay）同生命周期，不拆文件
GitStatusOverlay值对象Map<path, GitFileStatus>；不内嵌 FileNode，按 path joinstatus 频繁变 vs 树稳定，独立 ref 避免树重建
DetailPaneView值对象view ∈ {diff,preview}；filePath 非空SideDrawer 预览视图状态
FileNode vs TreeNode 统一（GAP-S6）：搭便车整合时统一命名 FileNode（shared/src/file-tree.ts），不再有 TreeNode/FileNode 两套。

关键决策

D-012 · 数据分离

FileNode 不含 gitStatus

张力：树节点内嵌 gitStatus vs 独立 overlay

决策：FileNode 不含 gitStatus；store 内 GitStatusOverlay 独立 ref；runtime 不合并 status（纯两步：file.tree 拿结构 + git.status 拿标注）

理由：status 频繁变 vs 树稳定，分离避免 status 变化触发整树重建

D-013 · REVISIT of D-008

IIgnoreReader 改纯函数

张力：ignore 包 port vs 仿 git-status-parser 纯函数豁免

决策：shared/ignore-parser 纯函数；IO 走 IFileExecutor.readFile。IIgnoreReader port 取消

理由：git-service.ts:26 实证 GitService 直接 import 纯函数豁免，D-008 声称对称却包 port 实际不对称

D-009 · 加载策略

懒加载

file.tree 顶层 + file.tree.expand 按需。全量加载在大项目慢/内存大；虚拟滚动过度工程。

D-011 · 状态机

宽松只守终态

不设显式转换表（单用户交互、前端单写者、无并发）。仅守 loaded 后折叠不丢数据 + error 可重试。
