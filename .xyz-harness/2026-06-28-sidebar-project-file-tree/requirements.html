§0核心语义变更：改动清单 → 全项目树

这是本次裁决的根本。现有 spec/draft 把 File View 定义成「当前 session 改动文件清单」，新设计改为「session 工作目录的完整文件树」。下表是逐维度的推翻对照：

维度旧（spec.md / draft-file-view.html）改动清单新（本稿）全项目树
数据源message.file_changes（agent 本次改的文件）file.tree（全树）+ git.status（标注层）
树内容仅改动文件（按路径分组）全部文件 · 未改动无角标 · 改动有角标 · 忽略项默认隐藏
tab 计数「文件 12 = 改动的 12 个」「文件 N = 项目文件总数」（标注里另列改动数）
空态单一：「这个会话还没有文件改动」两种：① 无 session（引导选 session）② 有 session 但目录空
切 session刷新为新 session 的改动文件刷新为新 session.cwd 的完整树 + 新 git 标注
后端依赖file_changes 通道（已就绪）file.tree 协议待建（runtime 无 file-service）+ git.status（已就绪）
⚠️ 推翻动作：定稿后需修订 docs/page-design/v3/sidebar/spec.md §视图切换（删「非全项目」、改计数语义）+ 重写 draft-file-view.html lede。本 demo 即新 draft 范本。

D-004 裁决：遵循 .gitignore 是全项目树可用性的前提。
不隐藏 → node_modules 数千文件淹没树 → 不可用。
.gitignore 是项目自带的标准 ignore 源，无需自造规则。
显示忽略项开关（眼睛图标，默认关）：
打开后忽略项以 斜体 + 半透明 显示，并标注项数（如 node_modules 2.4k 项，不递归展开）。
这是 escape hatch——开发者偶尔需要看 .env / 构建产物。

严格区分（handoff P0）：
本框 = 文件树过滤：实时、全树节点、无命令/符号/会话。
