TL;DR · 核心结论
  • 本质是技术流程编排——聚合 4 个异构数据源（命令注册表 / 项目文件 / 会话库 / recents），经统一子串匹配引擎过滤，按四类分组渲染，选中后分发跳转。
  • 三层架构（Interface / Service / Infrastructure），不做 DDD 四层——Domain 层无规则可放会是空壳（D-011）。
  • 不做 port（D-012）：4 数据源接口形状不同，强制抽象是伪 port；匹配引擎提为纯函数模块。
  • 松散状态机（D-014）：仅 closed↔open 是独立状态变量，open 内视图态全部由 Vue computed 派生。
  • 新建 search real domain 替换 mock（消除误导）+ 命令注册表扩展（应用命令区 + slash 命令区物理隔离）+ 匹配/跳转/recents 三模块提取。

2. 设计立场

核心计算是什么？ 技术流程编排——把 4 个异构数据源（命令注册表 / 项目文件 / 会话库 / recents）的查询结果聚合，经统一子串匹配引擎过滤，按四类分组渲染，选中后分发到对应跳转目标（命令执行 / 文件打开 / 会话切换）。

分层决策：三层架构（Interface / Service / Infrastructure）。无复杂业务规则 / 策略引擎，核心是「查询-匹配-渲染-跳转」的编排流水线。DDD 四层的 Domain 层会是空壳（无领域规则可放），违反复杂度归位（D-011）。

5. 状态流转

Status 枚举（浮层生命周期，只描述阶段）

closed → open（open 内含多个派生视图态）

open 内视图态（派生态，非独立状态变量，由 query/结果/loading computed 推导）：

  • recents（query 空）
  • query-results（query 非空 + 有命中）
  • empty（query 非空 + 无命中）
  • loading（查询中 >200ms）
  • error（查询/跳转失败）
  • type-filtered（Tab 切类激活时）

Reason 字段

无独立 Reason——error 态的原因由具体 api 调用的 catch 表达（toast 反馈），不进状态机。

松散状态机（D-014）：上述视图态是 Vue computed 派生（currentView = query空 ? recents : hasResults ? query-results : empty），不建显式转换表。closed↔open 是仅有的独立状态变量（props.open）。loading/error 是 transient 标志（ref + setTimeout/catch 驱动）。状态机详见上方主角图。

10. 挑战与决策

D-011 · 三层 vs DDD 四层

三层架构

张力：架构纯粹性（DDD 四层）vs 复杂度归位（核心无领域规则）

理由：核心是技术流程编排，Domain 层无规则可放会是空壳。

D-012 · port 边界

不做 port

张力：可扩展性（为数据源/匹配做 port）vs 伪 port 成本

理由：4 数据源接口形状不同，删/翻/挪证伪为伪 port；匹配策略稳定无替换需求。

D-013 · 领域模型深度

纯 DTO/值对象，无 aggregate

张力：建模完整性 vs 过度建模

理由：无领域不变式可守，建 SearchSession aggregate 是「装着 UI 逻辑的对象」反模式。

D-016 · 命令注册表归属

扩展 command store（全局应用命令区 + slash 命令区物理隔离）

张力：command store 已管 pi slash（按 sessionId 分区），应用命令是全局的（无 sessionId）。

理由：复用现有 SessionCommand 模型 + store 基建；应用命令与 slash 命令同属「命令」概念，内聚。

物理隔离（应对 S5）：两区失效语义不同——应用命令区静态注册（启动时一次性，无失效）；slash 命令区 per-session 动态（session 切换/创建时刷新）。store 内用两个独立 ref：appCommands: Ref<AppCommand[]> + slashCommands: Map<sessionId, SessionCommand[]>，不揉进同一响应式根。

特化决策

决策违反什么为什么合理触发变化怎么办
符号占位不建 SearchSource portD-012 不做 port 的通用规则符号是占位（无真实数据源），连查询都不发，无需 port补符号数据时新增 search domain 内一个 api 调用即可，仍不需 port
recents 用 localStorage 不走 runtime「数据应经 runtime」惯例recents 是纯前端偏好（D-007），与 settings 的 system 偏好同性质，不经 runtime如需跨设备同步再改，当前 YAGNI
