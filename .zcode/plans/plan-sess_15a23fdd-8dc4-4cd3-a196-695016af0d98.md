# 修复 bash trace 块 icon 被 flex 挤压 + 清理 bashExecution 死代码

## 问题根因

`Block.vue` 的 trace header 行（L83-92）是 flex 布局，`argPath` span 带 `truncate` 但缺 `min-w-0`，导致 `min-width: auto` 解析为 content 宽度（nowrap 文本全宽），拒绝收缩。flex 算法转而压缩所有 `flex-shrink: 1` 的 item——包括没有 `shrink-0` 的 Wrench/ChevronRight SVG。

实测（Playwright 400px 容器）：
- bash 长 command → Wrench 从 12px 压缩到 **4.25px**，ChevronRight 从 10px 到 **3.53px**
- read 短 path → Wrench 保持 12px

附带发现：event-adapter / chat-message-effects / SystemNotice 里的 bashExecution 处理链路是**死代码**（pi 从不 emit `message_start{role:'bashExecution'}`，6077 个事件零命中），只被 mock 触发。一并清理。
