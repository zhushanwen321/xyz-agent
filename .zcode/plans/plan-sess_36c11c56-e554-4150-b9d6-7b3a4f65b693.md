## 改动概述

把 sidebar session 的删除从「modal 确认」改为「原地两段式确认」：点击删除按钮 → 按钮原地变红确认态 → 再次点击才真正删除。点击页面别处取消确认态。

## 改动文件

### 1. `packages/renderer/src/components/sidebar/SessionItem.vue`（核心改动）

**新增确认态逻辑（script）：**
- 新增 `const confirming = ref(false)` —— 本项是否处于删除确认态
- 新增 `onRemoveClick()`：
  - `confirming` 为 false → 设为 true（进入确认态），**不 emit**
  - `confirming` 为 true → `emit('delete', session.id)` 并 reset
- 重命名按钮点击时也 `confirming.value = false`（点重命名即取消确认态）

**template 改动：**
- 删除按钮去掉 `@click.stop="emit('delete', ...)"`，改为 `@click.stop="onRemoveClick"`
- 按钮样式根据 `confirming` 动态切换：
  - 默认态（现状）：`bg-surface text-muted ... hover:bg-surface-hover hover:text-danger`，图标 `Trash2`，title「删除」
  - 确认态：`bg-danger text-white border-danger hover:bg-danger hover:text-white`（红底白字），图标换成 `Check`（lucide），title「确认删除？」
- 确认态时**强制显示**按钮区（不依赖 group-hover）：按钮容器 `div` 的 class 从 `hidden group-hover:flex` 改为 `:class="confirming ? 'flex' : 'hidden group-hover:flex'"`，确保鼠标短暂移开后回来仍可见红按钮
- 确认态时隐藏重命名按钮（`v-if="!confirming"`），避免两个按钮并列造成歧义

**点击别处取消（Phase 1 答复「点击空白/别处」）：**
- `confirming` 为 true 时，在根 `.session-item` 上监听鼠标移出（`@mouseleave`）→ reset。理由：group-hover 区域就是该 session item 的范围，鼠标移出整项即等价于「点到别处/移到别处」，且不引入全局 document listener（避免多实例 split 场景 listener 翻倍问题，铁律 #2）。
- 补充 `@blur` 不可行（div 无 focus），用 mouseleave 是最轻量且符合本场景的取消信号。

**import 新增：** `Check` from `@lucide/vue`，`ref` from `vue`。

### 2. `packages/renderer/src/components/sidebar/Sidebar.vue`（移除 modal 链路）

- 删除 `import DeleteSessionDialog from './DeleteSessionDialog.vue'`（line 156）

- `onConfirmDelete`（line 229-231）删除（不再有 confirm 事件）
- `deleteOpen` ref（line 178）删除
- `targetSessionLabel` computed（line 181-183）删除（仅 DeleteSessionDialog 消费）

- **lint**：新增 `Check` import 和动态 class 需过 taste-lint（no-hardcoded-colors 用语义类 `bg-danger`/`text-white`，`text-white` 项目中已大量使用作为图标在彩色底上的前景色，符合现有约定）
