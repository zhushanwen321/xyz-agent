**版本**: v0.1 | **日期**: 2026-04-08 | **状态**: 已确认

## 1. 概述

**目标**：跑通"用户输入 → Agent 调用 LLM → 流式输出到前端"的完整链路。

**技术栈**：Rust + Tauri v2 + Vue 3 + Tailwind CSS + shadcn-vue + reqwest + tokio + rusqlite → JSONL

**不包含**：工具调用、SubAgent、上下文压缩、Domain Pack

## 3. 数据模型

### 3.1 存储方案：JSONL + parentUuid 链式

参考 Claude Code 的存储方案。

**存储路径**：`~/.xyz-agent/projects/<sanitized-path>/<sessionId>.jsonl`

每行一个 JSON 对象，通过 `parent_uuid` 形成链式结构。

### 3.2 TranscriptEntry

```rust
// models/transcript.rs
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum TranscriptEntry {
```

### 3.3 AgentEvent

```rust
// models/event.rs
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum AgentEvent {
    TextDelta { session_id: String, delta: String },
    ThinkingDelta { session_id: String, delta: String },
    MessageComplete {
        session_id: String,
        role: String,
        content: String,
        usage: TokenUsage,
    },
    Error { session_id: String, message: String },
}
```

### 3.4 JSONL 操作（db/jsonl.rs）

- `append_entry(path, entry)` — 追加一行 JSONL
- `read_chain(path) -> Vec<TranscriptEntry>` — 读取全部条目
- `build_conversation_chain(entries, leaf_uuid) -> Vec<TranscriptEntry>` — 通过 parentUuid 回溯构建对话链
- `read_head_and_tail(path)` — 只读前 64KB + 后 64KB（大文件优化）
- 文件锁防止并发冲突

### 3.5 Session 索引（db/session_index.rs）

- 扫描 `~/.xyz-agent/projects/` 下 `.jsonl` 文件
- 从 head/tail 提取 session 元数据（id、title、时间）
- 缓存在内存中，定期刷新

## 4. Rust 侧核心模块

### 4.1 LLM Gateway（services/llm.rs）

```rust
#[async_trait]
pub trait LlmProvider: Send + Sync {
    async fn chat_stream(
        &self,
        messages: Vec<MessageParam>,
        model: &str,
    ) -> Result<Box<dyn Stream<Item = LlmStreamEvent> + Unpin>, LlmError>;
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum LlmStreamEvent {
    TextDelta { delta: String },
    ThinkingDelta { delta: String },
    MessageStop { usage: TokenUsage },
    Error { message: String },
}

pub struct AnthropicProvider {
    client: reqwest::Client,
    api_key: String,
    base_url: String,
}
```

**关键特性**：
- `LlmProvider` trait 抽象，P0 只实现 `AnthropicProvider`
- 返回 `Stream<Item = LlmStreamEvent>`
- 内置指数退避重试（默认 3 次）

### 4.2 AgentLoop（services/agent_loop.rs）

```rust
pub struct AgentLoop {
    provider: Arc<dyn LlmProvider>,
    session_id: String,
}

impl AgentLoop {
    /// 持续循环，通过 channel 输出事件
    pub async fn run(
        &mut self,
        user_message: String,
        history: Vec<TranscriptEntry>,
        event_tx: mpsc::UnboundedSender<AgentEvent>,
    ) -> Result<TranscriptEntry, AgentError> {
        // 1. 构建 messages（从 history 转换为 API 格式）
        // 2. 调用 provider.chat_stream()
        // 3. 消费流式事件，通过 event_tx 发送 AgentEvent
        // 4. auto-continue：如果 stop_reason 不是 end_turn，自动继续
        // 5. 返回最终的 assistant TranscriptEntry
    }
}
```

## 5. 前端设计

### 5.1 核心数据流

```
用户输入 → invoke('send_message', {sessionId, content})
                ↓
    Rust: AgentLoop 运行 → mpsc channel → 桥接 task
                ↓
    Tauri Event: 'agent-event' (TextDelta / MessageComplete / Error)
                ↓
    前端: listen('agent-event') → useChat composable → Vue reactive state
                ↓
    MessageBubble 实时渲染
```

### 5.2 useChat composable

```typescript
export function useChat(sessionId: Ref<string>) {
  const messages = ref<Message[]>([])
  const streamingText = ref('')
  const isStreaming = ref(false)

  onMounted(() => {
    listen<AgentEvent>('agent-event', (event) => {
      if (event.payload.session_id !== sessionId.value) return
      switch (event.payload.type) {
        case 'TextDelta':
          streamingText.value += event.payload.delta
          break
        case 'MessageComplete':
          messages.value.push(/* ... */)
          streamingText.value = ''
          isStreaming.value = false
          break
        case 'Error':
          // 错误处理
          break
      }
    })
  })

  async function sendMessage(content: string) {
    isStreaming.value = true
    messages.value.push({ role: 'user', content })
    await invoke('send_message', { sessionId: sessionId.value, content })
  }

  return { messages, streamingText, isStreaming, sendMessage }
}
```

## 6. 错误处理与安全性

### 6.1 统一错误类型

```rust
#[derive(Debug, thiserror::Error)]
pub enum AppError {
    #[error("LLM API error: {0}")]
    Llm(String),
    #[error("Storage error: {0}")]
    Storage(String),
    #[error("Session not found: {0}")]
    SessionNotFound(String),
    #[error("Config error: {0}")]
    Config(String),
}
```

Tauri Command 统一返回 `Result<T, String>`，由 `impl From<AppError> for String` 自动转换。

### 6.2 API Key 管理

- 优先从环境变量 `ANTHROPIC_API_KEY` 读取
- 备选从 `~/.xyz-agent/config.toml` 配置文件读取
- 不硬编码，不存 JSONL

### 6.3 并发安全

- 每个 session 的 JSONL 写入使用文件锁
- 每次 `send_message` 创建独立 tokio task，多 session 可并行
- 同一 session 不允许并发写入（前端 streaming 期间禁用发送按钮）

## 8. 验收标准

用户在 Tauri 窗口输入文字，能看到 Anthropic API 的流式输出。具体：
1. 新建 Session → 输入消息 → 看到逐字流式输出
2. Session 列表能显示历史会话
3. 切换 Session 能加载历史消息
4. 长消息（>1000 字）流式输出不卡顿
5. API Key 缺失时有明确错误提示

## 9. Claude Code 源码参考

设计过程中参考了 [Claude Code 源码](https://github.com/leeyeel/claude-code-sourcemap)，主要借鉴：

| 借鉴点 | Claude Code 实现 | xyz-agent 对应 |
|--------|-----------------|----------------|
| JSONL + parentUuid 链式存储 | `sessionStorage.ts` | `db/jsonl.rs` |
| AsyncGenerator 流式模式 | `query.ts` 的 `async function* query()` | `services/agent_loop.rs` + `Stream` |
| Stream 事件桥接 | `Stream<T>` + 回调 | `mpsc::channel` → Tauri Event |
| LLM Provider 抽象 | `queryModelWithStreaming()` + 自动重试 | `LlmProvider` trait + `AnthropicProvider` |
| 依赖注入 | `QueryDeps` 类型 | `LlmProvider` trait 可替换 |
