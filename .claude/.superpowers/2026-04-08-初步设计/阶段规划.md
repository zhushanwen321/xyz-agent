# xyz-agent 阶段规划

**版本**: v0.2 | **日期**: 2026-04-08 (更新: 2026-04-09) | **状态**: 已确认

---

## 项目概述

构建通用型 Agent 引擎，采用 A+C 融合架构（单 Agent + 按需树形分解 + Domain Pack 插件）。

**技术栈**：Rust + Tauri v2 + Vue 3 + tokio + JSONL + SSE
**策略**：先做引擎核心，暂不绑定领域；第一阶段仅对接 Anthropic API

---

## 总览

| 阶段 | 名称 | 核心目标 |
|------|------|---------|
| P0 | 骨架与基础对话 | 跑通"用户输入 → LLM 流式输出到前端"完整链路 | ✅ 已完成
| P1 | 工具系统与上下文 | Agent 能调用工具完成实际任务，管理长对话上下文 | 🔄 设计中 |
| P2 | 多 Agent 与任务树 | 复杂任务自动分解为 SubAgent 协作，可视化任务树 |
| P3 | 完善与扩展 | MCP 集成、Skill 系统、Domain Pack 框架、OpenAI 支持 |

---

## P0：骨架与基础对话

**目标**：跑通"用户输入 → Agent 调用 LLM → 流式输出到前端"的完整链路。

**不包含**：工具调用、SubAgent、上下文压缩、Domain Pack

### 功能清单

| 模块 | 功能点 | 说明 |
|------|--------|------|
| LLM Gateway | Anthropic Provider | `chat(messages, tools, stream, model)` 接口，流式适配 |
| LLM Gateway | Provider 抽象层 | 定义 Provider 协议（ABC），仅实现 Anthropic |
| AgentLoop | 主循环骨架 | while True: 调 LLM → 处理流式事件 → 暂无工具调用时 break |
| AgentLoop | EventBus | 进程内事件总线，发布 agent_message / tool_call 等事件 |
| 通信层 | SSE Server | Python 端 SSE 端点，将 EventBus 事件推送给前端 |
| Tauri 前端 | 对话界面 | 消息列表 + 输入框 + 流式渲染，通过 SSE 接收事件 |
| Tauri 前端 | 基础布局 | 侧边栏（Session 列表占位）+ 主内容区 |
| 持久化 | SQLite 基础 | Session 表 + Message 表，对话历史存取 |

### 验收标准

用户在 Tauri 窗口输入文字，能看到 Anthropic API 的流式输出。

## P1：工具系统与上下文管理

**目标**：Agent 能调用工具完成实际任务，能管理长对话的上下文。

**不包含**：SubAgent、TaskTree、MCP、Skill、Domain Pack 框架、analysis 工具（推迟到 P2 随 SubAgent 设计）

### 详细设计文档

P1 按模块拆分为独立 spec，按依赖顺序设计：

| 模块 | Spec 文件 | 状态 |
|------|----------|------|
| ToolRouter | `2026-04-09-P1-toolrouter/spec.md` | 已确认 |
| 内置工具 | `2026-04-09-P1-builtins/spec.md` | 已确认 |
| AgentLoop 集成 | `2026-04-09-P1-agentloop-tools/spec.md` | 已确认 |
| ContextManager | `2026-04-09-P1-context-manager/spec.md` | 已确认 |
| PromptManager | `2026-04-09-P1-prompt-manager/spec.md` | 已确认 |
| DataContext | 待设计 | - |

### 功能清单（更新后）

| 模块 | 功能点 | 说明 |
|------|--------|------|
| ToolRouter | 统一工具协议 | Tool trait: name, description, input_schema, is_concurrent_safe, timeout, call() |
| ToolRouter | 并发调度 | ToolExecutor: safe 工具 join_all 并发，unsafe 串行 |
| ToolRouter | 权限检查 | PermissionContext 两层（global + session），deny 优先 |
| 内置工具 | Read | cat -n 格式，路径白名单，concurrent_safe=true |
| 内置工具 | Write | 覆盖写入，路径白名单，concurrent_safe=false |
| 内置工具 | Bash | tokio::process::Command，输出 100KB 截断，concurrent_safe=false |
| AgentLoop | 多轮循环 | stop_reason=tool_use 时执行工具 → 结果回 messages → 继续推理 |
| AgentLoop | SSE 解析扩展 | ToolUseStart/InputDelta/End + stop_reason 提取 |
| AgentLoop | TranscriptEntry 修订 | content 改为 Vec\<ContentBlock\>（嵌入 tool_use/tool_result） |
| ContextManager | 第一层压缩（50%） | 自动裁剪：工具结果超 N 轮→引用标记；重复数据去重 |
| ContextManager | 第二层压缩（75%） | LLM 摘要：较早对话由 LLM 生成结构化摘要替换原文 |
| PromptManager | 静态/动态分层 | 静态层含角色定义+工具描述+cache_control；动态层含 DataContext |
| DataContext | 数据追踪 | 追踪已获取/计算的数据，生成摘要注入系统提示词 |

### 验收标准

Agent 能通过工具读写文件、执行代码，长对话（>50 轮）不会爆 context。

## P2：多 Agent 与任务树

**目标**：Agent 能分解复杂任务，派生 SubAgent 协作，用户能看到全局任务树。

### 功能清单

| 模块 | 功能点 | 说明 |
|------|--------|------|
| SubAgent | Preset 模式 | 预定义模板（system prompt + tools），主 Agent 指定角色名 + 任务 |
| SubAgent | Fork 模式 | 继承父 Agent 上下文快照 |
| SubAgent | 权限控制 | 创建时指定 allowed_tools / forbidden_tools |
| SubAgent | 反馈机制 | feedback() 工具，子→父传递，上级在可暂停状态处理 |
| TaskTree | 树结构 | TaskNode：id, type, status, parent_id, children_ids, contract, metrics |
| TaskTree | 递归限制 | max_depth=20, max_width=10 |
| TaskTree | 状态机 | pending → running ⇄ paused → completed / terminated |
| 预算守卫 | BudgetGuard | token_budget（90% 阈值提示收尾）、max_turns、max_tool_calls |
| 持久化 | 任务树持久化 | SQLite 存储，运行时内存维护 + 定期持久化 |
| 前端 | 任务树 Tab | 树结构视图，节点层级/状态/进度展示 |
| 前端 | 执行日志 Tab | 实时工具调用、LLM 请求/响应日志 |
| 前端 | 用户干预 | 暂停/恢复节点、终止子树、查看节点详情 |
| ContextManager | 第三层压缩（90%） | 协商压缩：系统列出可丢弃大块，AI 决策 |

### 验收标准

用户输入复杂任务，Agent 自动分解为多个 SubAgent 执行，前端实时展示任务树状态变化，用户可暂停/终止子任务。

## P3：完善与扩展

**目标**：补齐剩余功能，打通插件化能力，完成设计文档中描述的全部特性。

### 功能清单

| 模块 | 功能点 | 说明 |
|------|--------|------|
| MCP 集成 | MCP Client | 从配置加载服务器列表，启动时连接，发现并注册工具 |
| MCP 集成 | 工具命名 | `mcp__{server}__{tool_name}` 格式 |
| MCP 集成 | 传输协议 | stdio 传输，SSE/WebSocket 预留接口 |
| Skill 系统 | skill_load 工具 | 按需加载 YAML 技能指令到上下文 |
| Skill 系统 | YAML 定义 | name, description, prompt, suggested_tools, estimated_turns |
| Domain Pack | 插件框架 | pack.yaml + tools/ + skills/ + templates/ + knowledge/ + config.yaml |
| Domain Pack | 热加载 | 运行时加载/卸载 Domain Pack |
| LLM Gateway | OpenAI Provider | 新增 OpenAI 兼容接口实现 |
| 前端 | 节点状态 Tab | 选中节点的 contract、轮次、token 消耗详情 |
| 前端 | 历史消息 Tab | 用户与 Agent 对话历史 |
| 前端 | 状态栏 | 目录、git 分支、上下文 load、provider/模型、token 速度 |
| 配置 | YAML 配置 | 全局配置 + 运行时覆盖 |

### 验收标准

能加载外部 MCP 服务器和 Domain Pack，OpenAI 和 Anthropic 可切换，GUI 展示完整的调试和管理信息。
