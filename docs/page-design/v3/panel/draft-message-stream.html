§3未决落点：视觉裁决resolved = 本稿已落 · deferred = 需上游

规则稿 §4 列的未决项，本稿全部裁决落定（①–⑥ 均 resolved）。⑤⑥ 由产品契约在本轮关闭：summary 恒存在、steer/followup 时机归 runtime。

① 流式光标态（AI 未停止）resolved
回合进行中：无已工作按钮，改用 working 态（脉冲点 + 实时计时 + 思考/工具计数 badge），流程无背景下划线展开，末条带光标。停止后才切到「已工作」折叠态 + 总结。见主区回合 #3。
② 用户/助手消息形态resolved
用户 = 靠右气泡（无标签行）；助手 = 背景融为一体（透明无边框无 Assistant 标签）。视觉权重：气泡 > 总结文字，符合「问→答」直觉。
③ 执行流程折叠resolved
「已工作 X分 + 思考×N + 工具×N」按钮点开向下展开执行流程（背景融合、下划线行分隔）。总结 + 文件列表恒显，作为稳定锚点。localStorage 按 session+回合记忆。见主区回合 #1/#2。
④ 工具失败形态resolved
错误挂在 tool 块上：失败的 tool 块整块渲染红框（danger 边 + 淡红底 + 错误码），结果随 streaming 流式显示（末尾光标）。不另起 turn 级红框、不自动重试——错误是 tool 的属性，不是回合属性。agent 是否继续取决于 runtime。见画廊 B。
⑤ 收尾 Output Text 契约resolved
产品裁决：turn 末尾的最终 text 块恒存在（runtime 保证输出）——它就是普通的 Output Text，仅因位于收尾位而固定不折叠。视觉层无需回退占位——.turn-summary 恒为 turn 的收尾 output text。概念见规则稿 §2 Output Text 卡。
⑥ steer/followup 入流resolved
时机归 runtime 接口层（steer / followup 是不同 API 调用）。视觉层（message-stream + composer）不追踪入流时机，只保证气泡形态与普通用户消息一致——见靠右气泡。
本稿不碰：① 完整 composer（→ draft-composer-states）；② Side Drawer Diff / ChangeSet Detail（→ draft-detail-pane，修改文件行的跳转目标）；③ 双 Panel 主从差异（→ workspace/spec.md）。这些是相邻叶子，此处只交付 message-stream zone 的交互态。

7 类消息块 · 视觉规范SPEC

展开态下 message-stream 由这 7 类块按真实时序排列。关键概念：没有独立「Summary」类型——turn 中产出与收尾的都是同一种 Output Text，差别只在渲染规则（中间产出折进执行流程 / 收尾位固定不折叠）。配色遵循 design-system：状态走色相（reasoning 紫 / tool 青 / success 绿 / danger 红），禁用「左色条+亮底卡片」slop。

类型何时出现渲染规则
User Message用户提交靠右气泡，始终可见，不参与折叠，无标签行
Output Text ★AI 产出正文一种类型·两种渲染：①中间产出→折进执行流程（下划线行）；②收尾位→固定不折叠，作回合焦点（accent 强调）。两者数据同构，仅呈现不同
Reasoning模型推理紫底斜体，归折叠条 N reasoning 计数；长块可单独再折叠
Tool Call工具调用+结果青色，结果嵌灰底 mono；归 M tool 计数。失败整块红框（danger 边+淡红底+错误码）+streaming，不自动重试——错误是 tool 属性
File Changes ★回合改动回合默认可见，A/M/D badge + 行数统计；点行→Side Drawer Diff
Steer / Followup已提交未入流靠右 pending 气泡，虚线 + 脉冲圆点；入流后与普通用户气泡同形态（时机归 runtime）
System系统提示配额/上下文/模型切换，灰底折叠。Error 非独立类型，挂 Tool Call 块渲染红框

折叠规则汇总RULE

判定原则：用户默认只关心结果（收尾 Output Text + File Changes），过程按需钻取。

块类型默认归属交互
User显回合起点—
Output Text · 收尾显回合末·焦点固定不折叠 · 复制
File Changes显回合末点行→Side Drawer Diff
Reasoning隐折叠条内展开回合·长块可单折
Tool Call隐折叠条内失败整块红框·streaming
Output Text · 中间隐折叠条内展开回合·正常列出
System隐顶部或折叠展开查看
折叠条出现条件：回合内存在 ≥1 个 Reasoning / Tool / 中间 Output Text 块。纯文字回答（无工具、无推理）的回合无折叠条，唯一的 Output Text 即收尾位，直接接 User。
折叠条文案：已工作 Xm Xs（恒显）+ N reasoning（有则显）+ M tool（有则显），三者皆为 mono Pill。
