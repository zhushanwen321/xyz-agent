项目级资源入口（决策 panel_header_gear）。从 Panel header 齿轮唤起右侧抽屉，跟着当前 session 的项目走。只读视图：展示当前项目实际生效的 agent / skill，顶部 Tab 切换 Agent / Skill 视图；每个资源按发现来源排序展示 badge，最优先标生效，同名时项目维度覆盖全局——强制目录 > 可选目录，可选内靠前覆盖靠后（ADR-0020 §1.1）。无配置控件——加载路径在全局 Settings 统一勾选（决策 global_configures_project）。

01 触发上下文 · header 齿轮

breadcrumb 三段保持静态文本（对齐 panel/draft-breadcrumb-popovers.html），齿轮作为独立按钮落在 header 右侧。当前项目存在项目维度资源或覆盖时，齿轮高亮 + 角标 = 项目维度生效条数。

02 抽屉结构

右侧 480px 抽屉，遮罩 + blur。头（标题 + 项目 pill + 关闭）→ 体（滚动：顶部 Tab 切换 Agent / Skill，entity 按 badge 链标生效来源）→ 底（只读说明）。下方即抽屉打开态。

视图语义 · 抽屉不存任何配置，仅展示当前项目实际加载的资源：每个资源按发现来源排序展示 badge：最优先的标生效（强制目录 > 可选目录，可选内靠前覆盖靠后，agent / skill 统一，ADR-0020 §1.1），其余为被覆盖的同名来源。全局来源 = 全局加载路径（强制 ~/.xyz-agent + 可选 ~/.pi · ~/.claude · ~/.agents + pi-install skill）；项目来源 = 项目加载路径（强制 .xyz-agent + 可选 .agents）——同名时项目来源的 badge 排在前并标生效，全局同名来源显示为被覆盖。要改加载哪些目录，去全局 Settings（agent / skill 两页，决策 global_configures_project）。agent 目录变更受 AgentRegistry 单例约束（会话启动时生效，切项目需重开，ADR-0020 §5 ui_limit），skill 不受此约束。
