1 L1/L2 视觉去点击态 · 静态文本只读字段 · 不参与导航
  • cursor: default：去除 cursor: pointer，光标不变 —— 视觉上读作「文本」而非「按钮」
  • 无 hover 背景：:hover { background: transparent }，仅保留轻微 color 微调（--text-secondary 不变）作为可读性辅助，不暗示可点
  • 无 chevron ▾：去掉「下拉」隐喻的视觉符号 —— chevron 是 action affordance 之一，缺它即读作静态文本
  • padding 保留：2px 4px padding 仍保留，保证和分支段对齐（视觉节奏不破）
  • text 可选中：用户依然可以双击 / 长按选中项目名 / 会话名复制路径 —— 这是「文本字段」行为，不是「按钮」行为
  • 和分支段的视觉差异：分支段是 monospace 字体 + accent soft 背景 pill（视觉权重明显高于 L1/L2），三段在视觉层级上明显分级，避免「为啥这个能点而那两个不能点」的困惑
2 分支 popover 通用规则位置 / 动画 / 触发 / 关闭
  • 位置：从分支 crumb 段正下方浮出（top: 50px ＝ main-header 42px + 8px 呼吸），靠右对齐到 main-panel 右边界（不挤 breadcrumb 后续段）
  • 动画：fade + 8px Y 偏移，200ms · --duration；关闭反向。不用 320ms（避免点击即时反馈变慢）
  • 触发：① 点击分支 crumb；② 全局快捷键 ⌘B（与 sidebar 收起 toggle 同键 —— ⌘B 是 chrome 级 toggle，不专属 popover：popover 已开时按 ⌘B 关闭 popover，popover 已关时按 ⌘B 开 popover；sidebar 同理。冲突规约见 §6 决策记录 idx 1 / 卡 E 矩阵）
  • 关闭：① 点击 backdrop；② esc；③ 点击 popover 外区域；④ 选择项后自动关。点击分支 crumb 不开第二个，关闭当前
  • popover 宽度：320px。三 popover 时代宽度 360/380/320 反映触发频率反序，单 popover 后固定 320 即可 —— 分支是微切换，最快最窄
3 切换分支 · 主操作 · 卡片 A / Bgit 状态前置 · ahead/behind 预防同步坑
  • 内容：分两组 ——「本地」（git 可见分支 + 状态）+「remote」（origin/*）。底部一行「+ 新建分支」是动作项（accent 色，区别于数据项）
  • 当前分支：✓ 前缀 + --accent-soft pill 背景（card A 第 1 项）—— 与 sidebar branch pill 同色系
  • git 状态：右侧 meta 字段显示 ——HEAD（当前）/ N ahead（领先 remote）/ N ahead · M behind（分歧需 rebase）/ merged（已合入 main，可清理）
  • 「ahead/behind」语义：让用户切分支前先看到需不需要同步—— 避免切到旧分支再发现代码不对
  • 搜索过滤（card B）：输入即时过滤本地 + remote，命中段字符高亮（accent soft 背景），无匹配时显示「其余 N 条被过滤 · 清空 显示全部」
  • 搜索空态：所有匹配 0 条时显示「无匹配分支 · 检查拼写 / 切换 remote」+「清空」动作
4 新建分支 · 次操作 · 卡片 C内联表单 · 不开新弹层
  • 触发：① popover 内点击「+ 新建分支」项；② 全局快捷键 ⌘⇧N（从 popover 外直接打开，跳过列表）
  • 为什么不开新 dialog：新建是分支 popover 的延展动作，不是独立任务（用户已经在分支上下文里思考分支了）—— 弹 dialog 会打断连续操作
  • 表单字段：① 名称（必填，monospace 输入，实时校验：仅允许 [a-z0-9/_-]、不以 / 结尾、不与现有分支重名）；② 基础分支（默认当前分支，可下拉选其他 —— 避免新分支基于已 merged 的旧分支）
  • 名称实时校验：合法 → 右侧 ✓（accent）；非法 → 右侧 ✗（danger）+ 行下错误提示（红字 11px）。非法时「创建」按钮 disabled
  • 创建后行为：默认「创建并切换」（breadcrumb 立即更新到新分支，main-panel 重新加载代码状态）；可选「仅创建」（保留当前分支，新分支作为后续切换目标）—— 此选项放在「创建」按钮的右侧下拉箭头里（避免主操作列表过载）
  • 取消：popover-foot 的「取消」按钮 或 esc；回退到「默认态」列表（不关闭 popover）
5 状态联动 · popover 不只是切换器切换 / 新建后 main-panel 怎么变
  • 切换分支：popover 关闭 → breadcrumb 分支 pill 文字更新 → main-panel 内容保留结构（不刷消息流）但重新加载代码状态（diff tab / file view 重新拉该分支的数据）→ 状态点可能变（运行中 → stopped）
  • 新建并切换：popover 关闭 → breadcrumb 分支 pill 文字更新为新分支 → 当前 composer 状态保留（用户可能正在写 prompt，新分支继续编辑）→ git 状态显示「1 ahead」（因为基于当前分支 fork）→ sidebar 的 branch pill 同步更新
  • 新建仅创建：popover 关闭 → breadcrumb 不变 → 新分支出现在下次打开 popover 的列表里（已 sort 到合适位置）
  • 关闭未选：用户开 popover 看一眼就关（点击 backdrop / esc）→ 无任何变更，breadcrumb 高亮也回退（active → 普通）
  • 错误处理：若切换后加载失败（git 错误 / 文件丢失），main-panel 弹错误 toast + 保留旧内容，breadcrumb 回退到切换前
