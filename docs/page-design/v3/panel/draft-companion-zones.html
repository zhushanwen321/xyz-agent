companion-zones · progress + git 状态细化

composer 上下两条伴生 zone 的状态机。progress-zone 三态纯只读（待办 / 进行 / 完成，不可点击——它是 agent 执行进度的反馈展示，用户不在此干预）。git-zone 四态（干净 / 已暂存 / 有 diff / 冲突），其中查看类操作（Diff / 解决冲突）点击统一进入 Detail Pane，其真身归 draft-detail-pane 设计，本稿不画 detail 内容。两 zone 与 composer 共享同一条视觉带——统一卡片语言、紧凑垂直间距，各自独立成卡。

设计裁决 · 开放项收口

handoff 标注的待决项
progress-zone 是否可点击 / 有阻塞态？
纯只读，不可点击，无阻塞态。progress 是 agent 执行进度的反馈展示，用户不在此干预；agent 需要裁决时走对话/澄清，不挂在 progress-zone 上。状态仅三态（待办 / 进行 / 完成），完成态自动收起为单行横幅（状态驱动，非用户点击折叠）。整区不提供任何 detail 入口。
裁决 · 纯只读三态
查看类点击（Diff / 解决冲突）去哪？
统一进入 Detail Pane。不画 Side Drawer Detail 抽屉，不复活已删的 Process Panel。本稿只保留按钮入口（Diff / 解决冲突），点击行为标注「→ Detail Pane」，真身归 draft-detail-pane 设计。git 操作类按钮（暂存 / 提交 / 取消暂存）仍在 zone 内就地完成，不进 detail。
裁决 · 详情归 Detail Pane
进度条粒度？
双层：summary bar（5px 总进度，进行=蓝渐变 / 完成=绿）+ todo 列表（每项独立 pct-tag + mark 状态）。完成态自动收起为内联单行条（文案 + 4px bar + 百分比），节省垂直空间。
裁决 · 双层粒度
git 冲突态视觉？
左 3px danger 竖条 + 右渐隐 danger soft 底 + 红 conflict pill。只保留一个主操作「解决冲突」（danger 色）→ Detail Pane。不整片染红、不堆多个按钮，冲突的「需手动介入」语义靠 pill + 竖条即可一眼识别。
裁决 · 克制强调
与 composer 的视觉连贯怎么做？
统一带 + 独立卡片。三 zone 共享卡片语言（同 bg-input / border / radius-lg / 左右边距），垂直 6px 紧凑成「带」；但各自独立成卡——因 composer 有独立 focus ring、progress/git 有独立状态变化，合并容器会让 focus 与状态语义打架。active panel 时三者边框统一获得 subtle 强化。
裁决 · 统一带独立卡

验收 P0 自检（对 handoff §6）

  • progress 三态（待办 / 进行 / 完成）+ git 四态（干净 / 已暂存 / 有 diff / 冲突）全画
  • progress-zone 纯只读：无阻塞态、无 toggle、无 detail 入口、不可点击
  • 两 zone 与 composer 视觉连贯：统一卡片语言 + 紧凑成带，不割裂
  • 空态明确：progress 无任务整区隐藏；git 干净压缩单行（隐藏暂存/提交）
  • 查看类点击（Diff / 解决冲突）统一进 Detail Pane，不画 drawer、不出现 Process Panel 旧概念
  • 冷蓝 token 一致（继承 design-tokens.md，状态色 soft 按 design-system pill 惯例补全）
  • 无废弃术语（Panel / progress-zone / git-zone / Detail Pane，无 ChatView / Process Panel / Side Drawer）
