Overview · 多会话鸟瞰

L1 独立 Region。当 session 数超过 Sidebar Session List 的舒适承载（日常 3–5），或需监控多个后台 agent 时进入。取向 = 统筹/监控（网格、信息密集、可筛选排序），区别于 Session List 的 导航/切换（紧凑、单列、常驻）。卡片复用 Session Item 信息原子，密度上调一档。

§4后台 agent 进度聚合 · flow-3 联动

有子 agent 在跑的卡片，嵌入进度聚合块（多 agent 并行条）。这是 Overview 相对 Session List 的核心增量——鸟瞰多个后台编排。

聚合块复用 flow-3 的多进度方案（单 session 进度 → 多进度聚合）。条色：运行中 accent、接近完成 success、卡住 warning、失败 danger。Overview 是唯一同时鸟瞰多 session 后台 agent 的地方——Session List 只显角标计数，不显进度。
