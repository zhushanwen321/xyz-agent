§5模型行 + thinkingLevelMap 三档 ★核心特异点

模型行是 Provider 与别菜单拉开差异的地方。thinkingLevelMap: Record<string, string|null> 描述该模型支持哪些思考档位，归为三种策略。列表态用色编 pill 一眼区分，编辑态点行变分段控件可切。这是 handoff §4 的核心交互。

三档色编与语义

策略 key静态 pill分段（编辑态）语义（支持档位）色编
all-levels全档
 off / low / medium / high / xhigh 全支持--info 青 · 最丰富
high-max高/顶
 仅 high / max（顶配推理模型）--accent 蓝 · 产品主推
on-off开关
 二值开关（非推理或轻量模型）--muted 中性 · 最简
色编裁决：handoff 原文「binary=agent 色、highmax=accent」。本稿将 high-max 绑 --accent（产品主推配置，与默认 thinking=medium 呼应）；binary 用 --muted 中性（二值最简，不抢视觉）；all-levels 用 --info 青（全档最丰富，与 reasoning 标记同色系但 pill/rea 形态不同可区分）。如产品要 binary 也带色，改 .tpill.binary token 即可，色编规则集中在 CSS 一处。

§8验收对照 · P0 + 待裁决

验收 P0（handoff §7）：
✅ API key 加密显隐（password + 👁 toggle，已设显 ••••，§6）
✅ 测试连接可见反馈（ok/err + 模型数，§3）
✅ 模型 thinking 三档可选且色编正确（全档/高顶/开关），列表态点 pill 就地切分段（§5）
✅ 默认 provider 入口可见（head 右侧「设为默认」按钮 / pill acc「默认」，全局互斥，§2）
✅ key 字段不外泄（真实加密属 config-service 后端，本菜单只演示显隐，§6）

对齐检查：
✅ 视觉 = ADR-0018 冷蓝暗色（token 照搬 design-tokens.md，非 PRODUCT.md 暖色）
✅ 控件原语全从 design-system 派生（Entity/Row/Card/Toggle/Segmented/Pill/Dot），无自造变体
✅ 校验分层 = form-validation 规范（字段级 FormMessage / 表单级 inline status，无 toast 替代）
✅ 数据模型 = shared/src/provider.ts（ProviderInfo.status 三值 / ModelInfo.thinkingLevelMap / cost 四字段）
待裁决 / 空缺：
① thinking 三档色编——handoff 原文「binary=agent 色」未定义「agent 色」token，本稿裁决 binary=--muted、high-max=--accent、all-levels=--info。如产品要 binary 也带语义色，集中改 .tpill.binary / .tseg button.on.binary 一处。
② cost 字段来源——ModelInfo.cost{input,output,cacheRead,cacheWrite} 四字段。本稿表格已移除 cost 列（只保留 4 列：模型/推理/上下文/thinking，聚焦模型能力维度），cost 留待「发现后用户手填」，进不进表格待定。
③ headers / authHeader——ProviderInfo 支持自定义 headers 与 authHeader 开关，本稿未给 UI（低频高级项），可在编辑 Modal 加「高级」折叠区。
④ modelOverrides——pi-provider-store 的 modelOverrides（每模型参数覆盖）未给 UI，同③归高级区。
⑤ 真实集成出口：test-connection / discover-models / CRUD 走 config-service + model-service；key 加密存储走 pi-provider-store。
⑥ 故障转移不实现——用户明确不做。默认 Provider 走 head 按钮（setDefault() 全局互斥），凭据区不再放默认操作，底部「默认与轮询」卡已移除。
⑦ 默认模型 = 全局唯一——用户裁决：跨所有 provider 互斥，入口在模型表格行右侧「设为默认」按钮。后端缺口：ProviderInfo / ModelInfo（shared/src/provider.ts）当前无 isDefault / isDefaultModel 字段，本稿 UI 层先行（PROVIDERS[].isDefault + models[].isDefaultModel），真实集成需后端补字段并落地全局单例存储。
关联交付：modal 骨架与公共横切（搜索/自动保存/退出恢复）见 draft-settings-shell.html；其余 4 菜单（Skill / Agent / Extension / System）见同目录 handoff-*.md，按优先级 Agent → Provider(本稿) → Skill → Extension → System 细化。
