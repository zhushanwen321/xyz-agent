System · 外观与基础设置

System 菜单（左导航第 5 项）per-menu 细化稿。5 菜单里最轻量的一个——全枚举与开关，无强校验、无空态、无实体列表。主用 模式 B · Setting Card（三块分组）+ 模式 A · Setting Row。字段、默认值、布局、交互逐字对齐真身 ~/Code/xyz-agent-workspace/main：SystemPane.vue + stores/settings.ts + shared/settings.ts（ThemeMode/ThemePreset）。核心看点：配色不是改一个 CSS 变量，而是改 <html> 的 data-theme+data-palette 属性——本页所有 accent 色随你切换实时变化。

§1核心 · 主题切换机制（属性驱动，非 JS 逐变量赋值）

真身 applyTheme() 做两件事：给 <html> 写 data-theme（system→matchMedia('(prefers-color-scheme: dark)') 解析）+ data-palette，再双写 localStorage（xyz-agent-theme / xyz-agent-palette）。不改 JS 变量——CSS 属性选择器 html[data-palette="terracotta"]{ --accent: … } 自动重算整套 token。下方控件就是真身那两个 select + swatch，切一下，本页所有 accent（标题徽章 / 按钮 / 选中环 / code）实时换色。

§4★待裁决（3 项决议 + ④ 已执行演示增强）+ 验收

真身 SystemPane 是「语言 + 外观模式 + 聊天显示 + 配色主题」；本 draft 按决议移除「聊天显示」整块（autoExpandThinking / autoExpandToolCalls / compactStreaming / standaloneTools 四字段，真身仍保留）。draft-settings-shell.html 的 System 占位区画过「字号(紧凑/舒适/宽松) + 关于(版本/检查更新)」——与真身不符。三项是否超出真身新增，需你拍板：

① 关于区（版本 / 检查更新）— 倾向新增（最小化：版本号 + 检查更新按钮）。shell draft 已铺垫、属「应用自身」语义、成本低。
② 快捷键 capture — 倾向先不做可编辑版。capture + 冲突检测 + reset-to-default 是独立工作量；若要做，先以只读列表（⌘, / ⌘K / Esc / ←→）呈现，可编辑 capture 留后续。
③ 字号档（shell 占位遗留）— 倾向不做，真身无此字段。
④ draft 默认 palette = cold-blue（品牌主色，非真身 10 palette 之一）— 已执行：本 draft 初始 data-palette="cold-blue" + 选中 cold-blue swatch，使预览直接呈现项目实际主色冷蓝。真身代码默认 themePreset='neutral' 不改（§4 字段表保留真身默认，加 ★ 标注）。若要让真身也默认冷蓝，需新增 cold-blue 到 SystemPane.vue palettes[] + 改 settings.ts 初值。
