Flow 3 · 主 Agent 编排多子 Agent

护城河 flow。用户下达大任务，主 Agent 拆解派发多个子 Agent 并行执行，实时聚合进度，按需 steer/中断，汇总合并。本 flow 是 Flow 2（单 agent 串行）的并行编排升级。不出现 Process Panel（v1 已删）——子 agent 编排走 Side Drawer + progress-zone。

S5steer / 中断单个子 agent只影响该子

对运行中的子 agent 下达 steer（调整方向）或中断。steer 经 composer steer 态下发，只影响目标子 agent，不动主 agent 和其它子。

composer · steer 态（目标 auth-oauth）
auth-oauth
Shift+Enter 换行· ⏸ 中断此子 agent· Esc 退出 steer
steer ≠ 中断：steer = 调整方向（子 agent 继续跑，吸收新指令）；中断 = 标 stopped（保留已完成部分，停）。两者都只作用于目标子，主 agent 和其它子不受影响。
