④ 待你校验的存疑点

下列是链路里我拿不准、或上游文档留白、或需要你拍板的点。逐条过一遍，决定是「照此推进」还是「改」。

结构层存疑

  1. Sidebar 是「容器」还是「单列表」？✅ 已裁决为容器（见 sidebar/spec.md「多视图容器」）。draft-session-item / draft-file-view 各成独立 draft（✅ 已完成）。ui-skeleton.md 正文 ChatView 旧名替换待批量执行。
  2. Overview 形态？✅ 已裁决：按 handoff 定为独立 L1 Region + 卡片网格（非 4 模块拆分）。见 overview/spec.md + draft-overview.html。
  3. settings/ 要不要深化？当前标「待评估」。如果不进 v3 范围，建议从树里移除避免占位。

归属层存疑（panel/spec.md 已标未决项）

  1. Side Drawer 归 Overlays 还是 Workspace 常驻？当前归 Overlay。双 Panel 模式下它会覆盖对侧——是已知限制，v1 浮层方案。要不要现在就设计「内嵌化」？
  2. Process Panel 触发与位置？SubAgent 并行才出现，还是 Workspace 级常驻？需独立 deepening。
  3. 全局工作区操作的家（新建会话 / split 布局切换）？workspace spec 删了 main-header，待定归 Sidebar 顶部还是 panel-header。影响 split 触发入口。
  4. composer 的 @浮层算 composer 内还是 overlay？当前在 composer 6 态里（③）。它浮在 composer 上方，行为接近 popover——是否该提到 overlays/？

内容层存疑

  1. message-stream 8 类块是否齐全？当前：user / assistant / tool / reasoning / changeset / subagent / error / system。漏没漏？比如「引用上下文卡」「分叉点标记」要不要算独立类。
  2. composer 6 态是否够？当前：空 / 输入中 / @浮层 / 附件 / 发送中 / 停止。多行展开高度、模型切换、上下文指示器没单列——是合进状态还是独立。
