2 容器四态设计要点术语裁决后的结构性重排
  • 会话列表 A（Session List · 默认子视图）：时间分组 + 会话项四要素。今天 展开并演示 hover 操作，其余分组折叠成「N 个会话」条。hover 会话项 → 右侧 time 隐藏、2 个方形按钮（重命名 / 删除）浮现，按钮再 hover 显示名称 tooltip。
  • 文件视图 B（File View · 子视图）：点 segmented tab「文件」切换。目录默认全部折叠，左侧 > 为折叠标记（折叠 >、展开旋转成 v）。展开每层缩进仅 10px，防嵌套目录横向溢出。文件 icon 按扩展名着色（.html 青 / .md 紫，刻意避开 M/A/D 状态色）。保留 M/A/D，不显示行 diff。文件范围限定在当前 active session 的改动（顶部小字标会话名 + 分支）。
  • 收起态 C（Collapsed · 容器整体隐藏）：完全隐藏 Sidebar 容器，Workspace 占满全宽。工作区顶栏放 3 按钮：展开侧栏 / 后退 / 前进（方形 icon + hover 名称 tooltip + 快捷键），收起后导航能力不丢。唤回靠两路：① 顶栏「展开侧栏」按钮；② ⌘B toggle；③ 左缘细条 hover 浮出按钮。视图切换 tab 不进顶栏——收起后无需切子视图，恢复后回到隐藏前的 tab 状态。
  • 空状态 D（Empty · 数据空态）：按用户要求保持空——无虚线 icon、无引导文案、无新建按钮。子视图区仅一行极淡「暂无会话」占位，nav「新建任务 ⌘N」即入口。tab 计数归零（会话 0 / 文件 0）让用户从计数即知空。这是数据空态，与 A/B 的视图切换正交。
3 容器视图切换机制Session List ↔ File View · segmented tab
  • 分层结构：Sidebar 容器自上而下 = Brand → 主操作 nav（新建 ⌘N / 搜索 ⌘K）→ 视图切换 tab（会话 | 文件）→ 子视图区（Session List 或 File View）→ 用户区。主操作是动作（点即触发），视图切换是状态（点即停留）——二者性质不同，故分层而非混在 nav。
  • 切换触发：① 点 segmented tab；② 快捷键 ⌘1（会话）/ ⌘2（文件）（待定，见遗留）。两子视图互斥，同时只显一个。
  • 状态保持：tab 选择记入 Sidebar 容器状态，切换 session / 收起再展开 / 页面刷新均恢复到上次 tab。File View 的目录展开状态同理持久化。
  • tab 计数：每个 tab 右侧带小字计数（会话 N / 文件 M），让用户不切换即知规模。File View 计数 = 当前 active session 的改动文件数（非全项目）。
  • active session 联动：切换 session（点 Session Item）时，若当前在 File View，文件列表自动刷新为新 session 的改动；若在 Session List，则高亮迁移到新 session 项。
4 主操作 nav + 窗口快捷键新建 / 搜索 · ⌘B ⌘[ ⌘]
  • 主操作 nav：两项——新建任务 ⌘N / 搜索 ⌘K。原「文件」项已升级为视图切换 tab，不再占 nav 位。「搜索」是触发入口，点按或 ⌘K 弹出独立 Overlay（搜索浮层），浮层本体设计见 overlays/draft-search-modal.html，本稿不画。
  • 窗口快捷键：顶部 chrome 三按钮 hover 显示绑定——收起 ⌘B / 后退 ⌘[ / 前进 ⌘]。后退前进用于会话与历史导航（纯 CSS ::after tooltip，无需 JS）。
