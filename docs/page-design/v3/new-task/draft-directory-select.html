§5与 File View 树内搜索的严格区分（P0）

两条搜索不可混用：① 本 popover（directory select）= 全局 workspace 列表，跨项目，弹层形态；② File View 树内搜索 = 当前 active session 改动文件的树过滤，sidebar 内联，无浮层。 入口 / 范围 / 形态 / 结果全不同，详 overlays/spec.md §严格区分。

§6实现要点

  • 锚定算法：position:absolute; bottom: calc(100% - 10px); left:50%; transform:translateX(-50%)。popover 水平居中对齐 chip 中心，垂直方向在 chip 上方。当 chip 距视口顶 < 300px 时改向下展开（fallback 待实现）。
  • 三角指 dis：用 ::after 旋转 45° 方形 + 两条 border 模拟，避免额外 SVG 节点。位置 bottom:-5px; left:50%。
  • 扁平化跨组导航：搜索空时按 keyboard ↑↓ 在「列表 5 项 + 动作 2 项」间扁平化移动（不分组，selIdx 一个变量）。
  • chip ↔ popover 状态联动：popover 打开时 chip 走 .open（--surface-2 底 + inset 1px accent ring），关闭后 chip 回默认。两者状态机由 popover 的 open 状态统一控制，不重复判断。
  • composer 焦点保持：popover 打开/关闭不触发 composer blur，用户可继续输入，这是 v3 局部浮层的硬约束（区别于 search modal 这种全局浮层）。
  • 数据源：~/.xyz-agent/recent-workspaces.json 持久化最近 10 个（v1 限制）。v2 接 LSP 后端实现 workspace index 模糊搜索。
  • 权限：选目录后若该目录是 ~ 之外的位置，触发 macOS 隐私权限请求（~/Library/Mobile Documents/com~apple~CloudDocs/ 之类），首次需要走系统的「Files and Folders」授权弹窗。
