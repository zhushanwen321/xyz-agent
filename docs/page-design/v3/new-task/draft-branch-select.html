§2边缘态 · dirty 分支切走 · inline 二次确认条

用户在 popover 内选到 另一个 dirty 分支，或当前 dirty 分支有 2 个未提交文件，popover 关闭前弹行内确认条。这是 v3「dirty 切走必须告知」的硬约束——v1 选「留在工作区」（不自动 stash，git 自然行为，不主动制造用户没要求的副作用）。

§3五态 · 节奏

状态触发行为
A 打开 · 搜索空点 branch chipfocus 搜索框，显示当前仓库所有本地分支（按最新活动排序）
B 输入 · 即时过滤键入字符无 debounce（即时），列表收缩到命中项
C 键盘导航↑ ↓扁平化跨组（分支 + 动作）
D 切到 dirty 分支选中另一 dirty 分支弹 inline 二次确认条；不立即切，等用户点「切走」
E 关闭Esc / 点 chip 区域外 / 选中后确认popover 收起，chip 更新为新值

§4键盘契约

键行为边界
Esc关闭 popover有 inline 确认条时：先关确认条（再按 Esc 才关 popover）
↑ ↓扁平化跨组移动到顶 / 底循环
Enter确认当前选中分支 → 触发 dirty 检查 / 切分支；动作 → 触发动作
Tabv1 不切组（仅一组）v2 接多组（本地/远程）时启用

§5实现要点

  • 数据源：git branch --list（本地分支） + git status --porcelain（dirty 检测）。v1 不接远程分支（-r），避免 v1 范围爆炸。
  • dirty 检测成本：git status --porcelain 每次开 popover 跑一次（< 50ms），不缓存（cache 失效场景太多，git 状态是高频变化的状态）。
  • 排序：按 reflog 最新活动排序（git reflog show --all），不是按字母。当前分支永远置顶。
  • 宽度 420px：比 directory 略宽（420 vs 380），因为 refactor-coding-workflow-design 这类长分支名在 380 下会换行。统一宽度便于 popover 间切换的视觉稳定。
  • inline 确认条：从 popover 列表上方插入，不替换列表——用户可继续浏览其他分支。点击「取消」关掉确认条回到「选另一 dirty」状态。
  • chip ↔ popover 状态联动：与 directory popover 同源。chip .open 加 --surface-2 底 + inset 1px accent ring。
  • composer 焦点保持：popover 打开/关闭不触发 composer blur，用户可继续输入。
  • 权限边界：本地分支切换是 git 操作，需要工作目录权限。如果当前 directory 权限是「沙箱」级别（perm 不是「完全访问」），dirty 切走按钮 disabled + 提示「当前权限不允许切分支」。v1 简化为仅支持「完全访问」模式。
