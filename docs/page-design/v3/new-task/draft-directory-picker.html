§3modal 与 popover 的层级关系

层级形态z-indexbackdrop关闭时落点
workspace / sidebar / composer常态auto无—
directory popover / branch popover局部浮层50无（不模糊）composer 保留 focus
系统目录 picker / 创建分支 modal居中 modal201（>popover）有 · 模糊背景落回上一 popover / landing
系统 toast短暂反馈300（>modal）无自动消失
层级规则：modal 进栈时不关闭 popover（popover 在 modal backdrop 之下，被遮罩盖住），关闭 modal 后 popover 自动可见。 popover 不模糊（不打破主任务流），modal 必模糊（强制聚焦到 modal）。

§4实现要点（合并 §1 + §2）

  • backdrop：rgba(7,7,9,0.55) + backdrop-filter:blur(10px)，与 settings/draft-settings-shell.html 同源；点击 backdrop 关闭 modal（与 Esc 等价）。
  • modal 阴影：box-shadow: 0 0 0 1px rgba(0,0,0,0.3), 0 24px 70px rgba(0,0,0,0.55)——描边 + 浮起阴影，与 Card-Elevated 手法一致。
  • 系统目录 picker 实现：v2 自绘 = Electron dialog.showOpenDialog fallback + 内部 file list state（fs.readdir 懒加载）。macOS 隐私权限：选 ~/Library/Mobile Documents/ 之类需先在系统设置授权「文件和文件夹」。
  • picker 列表行高：5px 12px padding → 行高约 24px（与 sidebar session-item 同高），保持列表密度一致。
  • 排序箭头：仅当前排序列画向下箭头，其他列不画——避免视觉噪声。
  • 创建分支 IPC：renderer → runtime → git checkout -b <name>，runtime 回 success/fail，失败时 input 转 has-error + 错误文案（与已有错误态同源）。
  • 提交 spinner：用 border-top-color:#fff 配合父按钮的 accent 底，无需额外色 token；动画 0.8s linear infinite。
  • 无障碍：两个 modal 都 role="dialog" + aria-modal="true"；打开时 trap focus 在 modal 内，关闭后焦点还给触发元素（popover chip）。
  • v1 范围说明：选目录 picker 不实现「创建新文件夹」动作（避免 v1 范围爆炸），只走「打开已有目录」单动作；创建分支只支持基于 HEAD，不接 git ref picker / tag / commit SHA。
