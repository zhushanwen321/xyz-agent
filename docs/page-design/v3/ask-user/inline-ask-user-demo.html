ask-user 富交互渲染 — 覆盖 composer 位置

ask-user 出现时覆盖 composer 位置（composer 消失，输入禁止），不是浮在 body 上的全屏遮罩 modal。 对话历史全程可见。字体统一：tab / 标题 / 选项 label 均 13px，description 用 muted 色弱化一档建立层级。倒计时暴露 5min 超时。

实现要点
渲染位置：Panel.vue 的 composer-band（L79）。 ask-user 期间 v-if 互斥——有 pending ask-user 请求时渲染 AskUserOverlay，否则渲染 Composer。两者不共存，composer 自然消失、输入禁止。
数据源：复用 useExtensionUI(sessionId) 的 queue，但改为 per-panel（每个 panel 监听自己的 sessionId），不是现在的全局 singleton。 现状 ExtensionUIDialog 用 useExtensionUI(focusedSessionId) 只跟焦点 session——inline 后每个 panel 各自渲染自己的 ask-user，split 模式下两个 panel 可同时各有一个。
需改动的点：
1. AskUserOverlay 从 ExtensionUIDialog 移出，挂到 Panel.vue 的 composer 位
2. useExtensionUI 从模块级单例 queue 改为 per-sessionId 分区（或新增 per-panel 取数）
3. confirm / select / input / editor 简单原语是否也 inline？建议仍走 modal（紧急确认该抢焦点），只有 ask-user 走 inline
