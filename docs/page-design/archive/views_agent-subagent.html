### Issues Found (4)

**1. [HIGH] Unhandled promise rejection in handleMessage**
L283: `handleMessage` is async but the `ws.on('message')` callback
doesn't catch rejected promises. If `this.pool.sendMessage()` throws,
the error is silently swallowed.

**2. [MEDIUM] Type-unsafe payload casting**
L368-370: Three consecutive `as string` casts without validation.
Malformed WS messages will cause runtime errors.

**3. [MEDIUM] Missing session cleanup on disconnect**
`ws.on('close')` removes the session from the map but doesn't
abort in-flight pi RPC calls.

**4. [LOW] console.log in production code**
L370: Debug logging should use a proper logger with levels.

### Suggestions
- Add try/catch around handleMessage or use .catch()
- Validate payload shape before casting with zod
- Implement request cancellation on WS disconnect

审查发现了 4 个问题，其中 1 个高优先级。建议优先修复 handleMessage 的未捕获异常。
