SendModeStatusBar — 三种模式提示设计 Demo

清晰告诉用户每种模式怎么触发、什么时候自动切换

方式 A · 简洁+快捷键标记

Send · Enter 直接发送
Steer · ⌘+Enter 中断并发送
Queue · ⌥+Enter 排队 AI 回复中自动
说明：每个 mode 只显示自己的快捷键和触发条件。Queue 模式加了 "AI 回复中自动" 标签。
需要新增 ⌘+Enter 作为 steer 快捷键（当前没有快捷键用于 steer）。

方式 B · 全部快捷键始终显示

Send · Enter 发送   ⌥+Enter 排队   ⌘+Enter 中断
Steer · ⌘+Enter 中断并发送
Queue · ⌥+Enter 排队 AI 回复中自动   要中断？用 ⌘+Enter
说明：send 模式下也显示 queue 和 steer 的快捷键，方便用户发现。
始终可见所有快捷键，降低学习成本。

方式 C · 极简，仅当前模式快捷键

Send · Enter
Steer · ⌘+Enter
Queue · ⌥+Enter auto
说明：极简风格，只有当前模式的快捷键。
需要用户 hover 或点击查看详情（可以加 tooltip）。

推荐方案

  • 方式 A：简洁清楚，每个 mode 独立显示快捷键 + 触发条件
  • 增加 ⌘+Enter（macOS）/ Ctrl+Enter（Win/Linux）作为 steer 快捷键
  • Queue 的 "AI 回复中自动" 标签用 tag--auto 样式，表示不用按住 Alt
  • SendModeStatusBar 接收一个新的 prop isStreaming，用来显示自动标签 & 调整 queue 提示

完整键盘映射

Enter   普通发送  （非 streaming 时）
Alt+Enter   Queue 排队  （强制 follow_up，不打断 AI）
⌘/Ctrl+Enter   Steer 中断  （建议新增，强制打断 AI 并发送）

当 AI 正在回复时，Enter 自动走 Queue（无需按 Alt）。
