      How It Works — Full Mechanism

      ① npm

      • Auto-detect plain name vs npm: prefix
      • npm install to ~/.../npm/node_modules/
      • Validate isValidPiExtension() or rollback
      • On failure: classify error → show guidance
      • Add npm:source to settings.json

      ② Local

      • Copy dir → temp tmp/ext-scan-*/
      • Recursively discover pi extensions
      • Show list → user selects
      • cp -r selected → extensions/<name>/
      • Clean up temp

      ③ Git

      • git clone --depth 1 → tmp/ext-scan-*/
      • Run npm install for deps
      • Recursively discover pi extensions
      • Show list → user selects
      • cp -r selected → extensions/<name>/
      npm:pi-xxx → npm install → npm/node_modules/ → /local/path → cp → tmp/ → scan → select → github:user/repo → git clone → tmp/ → scan → select → extensions/<name>/ → ExtensionResolver → --extension args → pi Agent
      1
      User submits input from Settings UI
      Frontend sends WebSocket message per source type: extension.install / extension.installDir / extension.installGit
      2
      server.ts routes to ExtensionService
      Dispatches by message type. ExtensionService runs the corresponding install method.
      3
      npm install / clone / copy to temp
      npm: auto-normalize name (pi-xxx → npm:pi-xxx), run npm install, validate, persist.
      Local: copy directory to tmp/ext-scan-{ts}/ for safe scanning.
      Git: git clone --depth 1 to tmp/ext-scan-{ts}/, install deps.
      4
      Discover & present collection
      Recursively scan temp dir. Each subdirectory with a valid pi extension is a candidate. Frontend shows a checkbox list — user selects which ones to install. For npm: single extension, no discovery step needed.
      5
      Install selected into target directory
      cp -r each selected extension → ~/.xyz-agent/pi/agent/extensions/<name>/. Add to settings.json packages[] if needed. Clean up temp dir. ExtensionResolver.scanThirdPartyExtensions() picks them up automatically.
      6
      Loaded into pi on next session
      getExtensionPaths() → filter disabled → append file-type extension → --extension <path> arg on pi spawn → pi's jiti loader registers the extension.
      npm Error Classification & Guidance
      Package not found
      404 on npm registry. → Check spelling and scope. Try npm view <pkg> first.
      Not a pi extension
      Package exists but no pi manifest. → Verify pi field in package.json, or peerDeps matching pi.
      Network / npm error
      Timeout, registry unreachable, permission denied. → Check npm config and network. Try npm install manually.
      Source Precedence (dedup — higher wins)
      1. npm (package.json deps) → 2. user (CLI --extension) → 3. settings (npm: / git: sources) → 4. third-party (extensions/ dir) → 5. bundled
      Data dir: ~/.xyz-agent/pi/agent/
      Temp dir: ~/.xyz-agent/pi/agent/tmp/ext-scan-*/ — auto cleaned after install.
      Apply: Extensions apply on next pi session. Use /reload for hot-reload.
