§1核心 · 安装工作流状态机

Extension 与 Skill 最大差异：Skill 扫描本地目录，Extension 走 npm / 路径 / git 三种来源 + 后端发现 + 真实安装。三阶段：discovering（扫描/克隆）→ selecting（候选多选，仅 local/git）→ installing。npm 直装无 select 阶段。全程 60s 超时兜底。下方每卡一个态，可交互。

§5决策记录 · Plugins 归属

已决策：① 保留 Plugins 独立菜单（2026-06-19，用户裁决）。Extension 不集成 per-item config——维持「安装/启用/卸载」三动作，MetaGrid 只读。

决策理由：Plugins 有独立的 per-item 配置表单（PluginSettingsForm.vue），与 Extension 的「远程安装 + 启用开关」职责不同。折叠进 Extension 需给 ExtensionInfo 加 configSchema 字段 + 展开态配置表单，过度耦合。Plugins 菜单作为第 6 项独立保留，本次不在 settings draft 队列内（用户单独实现）。
被否选项：② 折叠进 Extension（需加 configSchema，耦合过重）；③ 折叠进 Skill（语义不符——Skill 本地扫描、Extension 远程安装）。
验收对照（handoff-extension.md §7 P0）：
✅ 三 tab 安装来源都能触发工作流，进度态文案正确（§1 Stage A/B/D + 按钮文案 Cloning/Scanning/Installing）
✅ 候选多选 → 「安装选中 (N)」按选中数启用（§1 Stage C，可点行 toggle 看计数变化）
✅ 60s 超时有反馈（§1 Stage E，inline 错误）
✅ 卸载走确认 Dialog（§3，点 §2 卸载按钮真实弹窗）
✅ 启用开关即时生效（§2 toggle，extension.toggle；built-in disabled）
