▾ xyz-agent plugin 如何封装 pi extension
pi extension (外部, 运行在 pi 进程) - 通过 ctx.ui.setStatus({text, type}) 上报状态 - 例子: goal extension 上报 "Goal 3/20 | 2/5 tasks" - 数据格式: {text: string, type: 'info'|'warn'|'error'} xyz-agent plugin (内部, 运行在 Worker Thread) - plugin system 的 Worker 线程 - 通过 pluginContext.statusBar.update({key, label, ...}) - 例子: todo plugin 上报 "Todo 4/7" - 数据格式: StatusBarEntry {id, label, color?, priority?} 封装逻辑 (在 server.ts / event-adapter) 1. pi extension setStatus → 转换为统一 StatusBarEntry 2. xyz-agent plugin statusBarUpdate → 已是 StatusBarEntry 3. 两条流 merge 后统一 push 给前端 4. 前端 statusBarStore 按 source 标记 [PI] / [XYZ]
