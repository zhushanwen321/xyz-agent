Breaking Change
InputToolbar: 需支持 per-level display label
  • On/Off: off→"Off", xhigh→"On"
  • 其他策略: 保持原名

thinkingLevelMap schema:
Record<string, string | null>
→ 需扩展以支持 label，或由 InputToolbar 根据 map 推断

Option A: 在 map value 中编码 label
{ xhigh: "on" } — 用特殊值 "on" 作为 label
{ xhigh: "max" } — "max" 既是 label 也是 API 值
这意味着 On/Off 的 xhigh 值是 "on"，不是 null。

Option B: 独立 labelMap 字段
thinkingLevelMap + thinkingLevelLabels
两字段分离，更清晰但更复杂。

Recommend: Option A — On/Off 用 {minimal:null, low:null, medium:null, high:null, xhigh:"on"}，xhigh 的 value "on" 既当 display label 又当 API 映射值。但 "on" 不是有效 API level...

Option C (simplest): InputToolbar 根据 visible levels 数量推断 label
  • 只剩 off + 1 个 level → 显示 Off / On
  • 否则显示原名
不改 thinkingLevelMap schema。
