## Task 1: ContentBlock 类型 + TranscriptEntry content 改为 Vec

**Files:**
- Modify: `src-tauri/src/models/transcript.rs`

- [ ] **Step 1: 定义 ContentBlock 枚举**

在 `transcript.rs` 的 `TokenUsage` 之后、`TranscriptEntry` 之前添加：

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum AssistantContentBlock {
    #[serde(rename = "text")]
    Text { text: String },
    #[serde(rename = "tool_use")]
    ToolUse { id: String, name: String, input: serde_json::Value },
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum UserContentBlock {
    #[serde(rename = "text")]
    Text { text: String },
    #[serde(rename = "tool_result")]
    ToolResult { tool_use_id: String, content: String, is_error: bool },
}
```

- [ ] **Step 2: 修改 TranscriptEntry 的 content 字段**

User: `content: String` → `content: Vec<UserContentBlock>`
Assistant: `content: String` → `content: Vec<AssistantContentBlock>`

- [ ] **Step 3: 添加向后兼容的自定义反序列化**

使用 `#[serde(deserialize_with)]` 兼容旧格式：
- 旧格式 `content: "text"` → `vec![TextBlock { text: "text" }]`
- 新格式 `content: [{type: "text", text: "..."}]` → 直接反序列化

为 User 和 Assistant 分别添加辅助函数：

```rust
fn deserialize_user_content<'de, D>(deserializer: D) -> Result<Vec<UserContentBlock>, D::Error>
where D: serde::Deserializer<'de> {
    use serde::de;
    
    #[derive(Deserialize)]
    #[serde(untagged)]
    enum ContentOrString {
        Blocks(Vec<UserContentBlock>),
        String(String),
    }
    
    match ContentOrString::deserialize(deserializer)? {
        ContentOrString::Blocks(blocks) => Ok(blocks),
        ContentOrString::String(s) => Ok(vec![UserContentBlock::Text { text: s }]),
    }
}
```

对 Assistant 做同样处理。在 User 和 Assistant variant 的 content 字段上添加 `#[serde(deserialize_with = "deserialize_user_content")]`。

- [ ] **Step 4: 更新辅助方法**

`new_user()` → content 参数改为 `Vec<UserContentBlock>`，构造 `vec![UserContentBlock::Text { text: content }]` 的便捷方法保留。
`new_assistant()` → 同理。
`content()` 方法需要改为返回拼接文本（向后兼容）。

- [ ] **Step 5: 更新现有测试**

现有测试中的 `content: "hello"` → `content: vec![UserContentBlock::Text { text: "hello".into() }]`。
确保所有测试通过。

- [ ] **Step 6: 添加新测试**

测试旧格式反序列化兼容：`{"type":"user","content":"hello",...}` 能正确反序列化为 `vec![TextBlock]`。
测试新格式：`{"type":"assistant","content":[{"type":"tool_use","id":"t1",...}],...}`。

pub trait Tool: Send + Sync {
    fn name(&self) -> &str;

}
