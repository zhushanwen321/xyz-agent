## 用户提出的优化点分析

### 1. 每条消息右侧的操作菜单

**当前状态**：完全没有。MessageBubble 只有点击 body 的事件委托（复制代码块、展开折叠、打开外部链接）。

**分析**：需要 overflow menu 触发点，通常放在气泡右上角，悬停时显示 `...` 按钮，避免视觉噪音。

### 2. 复制消息功能

**当前状态**：只有代码块级别的复制。没有"复制整条消息"的功能。

**分析**：
- **单条复制**：从 `message.content` 取 markdown 源码
- **批量选择复制**：需要引入选择模式
- **复制格式切换**：markdown 源码 vs plain text
- **实现位置**：操作菜单是自然的入口

### 3. Navigate / Fork / Clone

**当前状态**：tree store 已内置 navigateCapable、setLeafId 等基础设施，SessionTreePanel 中已有 navigate 实现，但 MessageBubble 层面没有暴露这些操作。

**分析**：
- **Navigate**：跳转到树中某个指定 entry。通过 `entry.navigate` RPC
- **Fork**：从当前点分叉出新分支。需创建新 session
- **Clone**：克隆整个 session

### 4. 分支/树可视化 per 消息

**当前状态**：SessionTreePanel 已有完整树渲染，但消息气泡上完全没有分支提示。

**分析**：气泡底部右边缘的 pill 指示器是最轻量的方案。BranchTab 类型已存在可直接复用。

## 设计决策问题

### 消息操作菜单的位置逻辑

- 用户消息（右对齐）→ 菜单右上角=页面右侧边缘
- 助手消息（左对齐）→ 菜单右上角=气泡的右侧
- 统一规则：菜单始终在气泡右上角，贴气泡边框
- 悬停显示，减少视觉噪音

### 分支信息的呈现模式（三级方案）

| 方案 | 描述 | 适用场景 |
|---|---|---|
| Pill 指示器 | 气泡底部右边缘显示 "↳ 3" 的 pill，点击展开分支列表（Dropdown） | 轻量展示 |
| 分支 tab 行 | 气泡下方直接显示分支 tabs（类似浏览器 tab） | 需要可见性 |
| 侧边树 miniview | 消息区域左侧固定窄条，显示精简树状图 | 全局树感知 |

### 复制格式切换的交互

- 默认 markdown（开发者使用习惯）
- 操作菜单中两个选项："复制 Markdown" / "复制纯文本"
- 批量模式下顶部浮动栏提供格式 toggle

### 批量选择模式的入口

- 长按/右键消息 → "选择消息"
- 从 header 工具栏进入选择模式
- 按住 Shift 点击多选
- 顶部浮动栏："已选 N 条消息 · 复制 Markdown | 复制纯文本 | 取消"

## 已发现的设计问题

| 问题 | 严重度 | 描述 |
|---|---|---|
| `--user-bubble-bg` 只在 dark theme 定义，light theme 缺失 | **P1** | 浅色模式下用户消息气泡可能透明背景 |
| CSS accent 为蓝色 (hue 195)，DESIGN.md 指定赤陶色 (hue 28) | **P2** | 主题与设计文档不一致 |
| `SlashMenu.vue` 侧边条纹 (border-left: 3px) | **P2** | AI slop 反模式 |
| 测试文件 em dash 过度使用 | **P3** | 影响小 |

## 第一轮优化范围

基于高影响力、低复杂度的筛选原则，第一轮覆盖：

1. **消息操作菜单**（P0）— 每条消息右上角的 `...` 菜单，包含复制、操作入口
2. **复制消息**（P0）— 单条消息复制（markdown + plain text 两种格式）
3. **分支指示器 per 消息**（P0）— 气泡底部显示分支数 pill
4. **回到顶端/到底部**（中优先级）— 消息区域的浮动导航按钮
5. **修复 `--user-bubble-bg` light theme**（P1 bugfix）— CSS 变量补全

后续轮次：
- **第二轮**：Fork/Clone 操作 + Navigate 集成 + 消息搜索
- **第三轮**：批量选择/复制 + 摘要目录导航 + 编辑已发送消息
