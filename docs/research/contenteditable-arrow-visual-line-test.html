contenteditable 软换行方向键行为验证

实验 1：纯文本软换行（复刻 ComposerInput DOM）

单 <div contenteditable> + 纯文本（无 <br>），窄宽度自动折行。
测试方法：点击输入框，把光标放中间行，按 ↑↓。 观察光标是否逐视觉行移动，以及「光标是否在第一行/最后行」的判定。
关键问题：浏览器原生 ↑↓ 在纯文本软换行时，能否逐视觉行移动光标？
这是一段很长的中文文本它会因为容器宽度限制而自动折行成多个视觉行显示在这里供我们测试方向键的光标移动行为是否能逐行移动
点击输入框并按方向键

这是第一行很长的文本它会自动折行

实验 3：Selection.modify('move', dir, 'line') 副作用探测

复刻当前 moveCaretVerticalOf 的行为：在当前光标位置执行 sel.modify('move', 'backward'/'forward', 'line')， 比较 container+offset 是否变化。
关键问题：这个 API 在软换行场景下是否移不动光标？ 注意：实验 3 会真的移动光标（副作用）。
